## 一、`MD5Util` —— 密码哈希(49 行)

> 📍 文件:[`util/MD5Util.java`](../../src/main/java/com/game/util/MD5Util.java) · [类 L14](../../src/main/java/com/game/util/MD5Util.java#L14) · [`md5()` L26](../../src/main/java/com/game/util/MD5Util.md5#L26)

### 全部代码(就一个方法)

```java
public final class MD5Util {                         // final + 私有构造 = 不可实例化
    private MD5Util() {}

    public static String md5(String plain) {
        if (plain == null) return null;
        try {
            MessageDigest md = MessageDigest.getInstance("MD5");  // L31,拿 MD5 摘要器
            byte[] bytes = md.digest(plain.getBytes(UTF_8));      // L32,算出 16 字节摘要
            StringBuilder sb = new StringBuilder();
            for (byte b : bytes) {                                 // L34,每字节转 2 位十六进制
                String hex = Integer.toHexString(b & 0xff);       //   b & 0xff:byte 当无符号
                if (hex.length() == 1) sb.append('0');             //   补前导 0(如 f → 0f)
                sb.append(hex);
            }
            return sb.toString();                                 // 32 位十六进制串
        } catch (NoSuchAlgorithmException e) {
            throw new RuntimeException("MD5 算法不可用", e);       // MD5 是 JDK 内置,正常不会缺
        }
    }
}
```

### 逐行关键点

- **`MessageDigest.getInstance("MD5")`** — JDK 内置的加密摘要工具,传入算法名拿到实例。MD5 把任意长度输入算成**固定 128 位(16 字节)**的摘要。
- **`md.digest(bytes)`** — 执行计算,返回 16 字节的 `byte[]`。`"123456"` → `[e1, 0a, dc, ...]`(16 个字节)。
- **`b & 0xff`** — Java 的 `byte` 是有符号的(-128~127),负数的 `toHexString` 会出一串 f。`& 0xff` 把它转成无符号(0~255),保证输出 2 位正十六进制。
- **补前导 0** — 一个字节是 0~255,十六进制是 00~ff。但 `Integer.toHexString(15)` = `"f"`(只有 1 位),补 0 → `"0f"`。16 字节 × 2 位 = **32 位十六进制串**。

### 怎么被用

```java
// UserDao.register:
ps.setString(2, MD5Util.md5(user.getPassword()));   // 存库前哈希
// UserDao.login:
ps.setString(2, MD5Util.md5(password));              // 登录时哈希后比对
```

密码**永远不存明文**——注册时存 MD5 哈希,登录时把用户输入也 MD5、两个哈希比对。DB 里存的是 `e10adc3949ba59abbe56e057f20f883e`(=MD5("123456"))。

> **局限**:MD5 已被破解(彩虹表秒查),生产应换 BCrypt + 随机盐。课设够用。详见 [已知问题 #1](../../docs/已知问题与后续优化.md)。

## 四、`SoundUtil` —— 纯代码合成音效 ⭐⭐(独立亮点)

> 📍 文件:[`util/SoundUtil.java`](../../src/main/java/com/game/util/SoundUtil.java)(320 行)
> 深入讲解见 [04 答辩准备 §1 SoundUtil](./04_代码精读_音效排行admin.md);这里按代码结构讲。

### 4.1 播放模型:单线路 + 队列 + 守护线程

> 📍 [常量 L52–81](../../src/main/java/com/game/util/SoundUtil.java#L52) · [`playLoop` L93](../../src/main/java/com/game/util/SoundUtil.java#L93) · [`ensureLine` L111](../../src/main/java/com/game/util/SoundUtil.java#L111)

```
shoot()/hit()/... ──offer──▶ playQueue(BlockingQueue)
                                  │ take()
                                  ▼
                          sound-player 守护线程
                                  │ ensureLine() 拿 sharedLine
                                  ▼
                          buildTone 合成 PCM → line.write
```

- **单条 `SourceDataLine`**(`sharedLine`,懒加载):所有音效共用一条音频线,避免反复 open/close 耗尽混音器线路。
- **`BlockingQueue<Runnable>`** + **守护消费线程**:音效任务丢队列,由专用线程串行处理(不阻塞 EDT/游戏循环)。
- **防抖**(`shoot` L139):两次开火音至少间隔 40ms。
- **队列上限**(`MAX_PENDING=2`):超了丢弃,防连射堆爆。

### 4.2 四个音效方法(L134–180)

| 方法 | 合成 | 听感 |
|---|---|---|
| `shoot()` L134 | `buildTone(600Hz, 40ms, decay=25)` + 防抖 + 队列上限 | 短促 pew |
| `hit()` L152 | `buildTone(150Hz, 60ms, decay=12)` | 低沉击打 |
| `hurt()` L165 | `buildSweep(400→200Hz, 120ms)` | 下滑痛感 |
| `gameOver()` L175 | `buildNotes(660,520,392,262Hz)` | 递降"完蛋了" |

每个方法先 `shouldSkip()`(静音/音量0 直接 return),再 `enqueue()` 入队。

### 4.3 合成方法:`buildTone` / `buildSweep` / `appendSample` ⭐

> 📍 [`buildTone` L227](../../src/main/java/com/game/util/SoundUtil.java#L227) · [`buildSweep` L256](../../src/main/java/com/game/util/SoundUtil.java#L256) · [`appendSample` L309](../../src/main/java/com/game/util/SoundUtil.java#L309)

**`buildTone`**(合成一段固定频率的音):

```java
for (int i = 0; i < samples; i++) {
    double t = i / SAMPLE_RATE;                    // 时间(秒)
    double env = Math.exp(-decay * t);              // 指数衰减包络(声音渐小)
    double atk = t < 0.005 ? t/0.005 : 1.0;        // 起音淡入(消除开头"咔"声)
    double rel = (剩余 < 0.005) ? 剩余/0.005 : 1.0; // 结尾淡出(消除连射爆音)
    double s = Math.sin(2 * Math.PI * freq * t);    // 正弦波
    appendSample(buffer, i, s * amp * env * atk * rel);  // 叠起来 → 写字节
}
```

一段音 = **波形(正弦) × 衰减包络 × 起音 × 释音 × 音量**。44100Hz/16bit/单声道。

**`appendSample`**(double 样本 → 16bit 小端序两字节):

```java
short val = (short) Math.round(value);        // double → short(钳制)
buffer[2*index]   = (byte)(val & 0xff);       // 低字节(小端)
buffer[2*index+1] = (byte)((val >> 8) & 0xff); // 高字节
```

> 详细原理(PCM / 采样 / 衰减 / 相位累加 / 小缓冲 / 淡入淡出修 bug)见 [04 答辩准备 §1](./04_代码精读_音效排行admin.md) 和 [音效修复复盘](../v1.0_审计修复/扩展_音效修复_射击干脆与松手即停.md)。

<details><summary>答案</summary>

1. `final` = 不能被继承(防止子类改行为);私有构造 = 不能 `new`(工具类不该有实例)。合起来 = **只能用静态方法,不能实例化也不能继承**。
2. 32 位十六进制字符串(`e10adc3949ba59abbe56e057f20f883e`)。
3. **不在**。注释明确写了「仅作用于本次运行」——`static` 字段存在内存,程序退出就没了。不持久化到文件/DB。
4. `UIManager.put(key, value)` 改 Swing 全局外观字典的默认值 → 所有组件自动跟随。一处改、处处生效。
5. `primary` 返回的是**自绘圆角暗色按钮**(`GameButton`),有 hover 效果、橙底白字;`new JButton` 是**系统默认按钮**(灰矩形,无 hover)。
6. EDT 是唯一管界面绘制的线程,`line.write` 会阻塞(音频缓冲满时)——如果在 EDT 里播音效,会卡界面。丢给守护线程串行播放,主循环立刻返回。
7. `val & 0xff` 取**低字节**(`&` 掩掉高位);`(val >> 8) & 0xff` 取**高字节**(右移 8 位再掩)。合起来 = 把一个 short(16 位)拆成两个 byte(小端序:低字节在前)。

</details>

### 8.1 `static {}` 块:类加载时读一次配置

> 📍 [`static {}` L26–40](../../src/main/java/com/game/dao/DBUtil.java#L26)

```java

public static Connection getConnection() {

        return DriverManager.getConnection(URL, USER, PASSWORD);
```

### 8.3 `close()` —— 倒序关资源

```java

    if (rs != null)   try { rs.close(); }   catch (SQLException ignored) {}
    if (stmt != null) try { stmt.close(); } catch (SQLException ignored) {}
    if (conn != null) try { conn.close(); } catch (SQLException ignored) {}
}
```

**倒序关**(先开的后关):结果集 → 语句 → 连接。每个**单独 try**:一个关失败不影响后面的(否则连接泄漏)。本项目 DAO 现多用 `try-with-resources` 自动关,这个 `close` 是给手动管理留的。

## 九、`UserDao` —— 用户增删改查(含事务)

> 📍 文件:[`dao/UserDao.java`](../../src/main/java/com/game/dao/UserDao.java)(~235 行)

### 9.1 `login(phone, password)` ⭐ PreparedStatement 防注入

> 📍 [`login()` L73–99](../../src/main/java/com/game/dao/UserDao.java#L73)

```java
public User login(String phone, String password) {
    String sql = "SELECT id, phone, password, nickname, role, create_time FROM user "
               + "WHERE phone = ? AND password = ?";      // ? 占位
    try (Connection conn = DBUtil.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
        ps.setString(1, phone);                           // 填第 1 个 ?
        ps.setString(2, MD5Util.md5(password));           // 填第 2 个 ?(密码先 MD5)
        try (ResultSet rs = ps.executeQuery()) {
            if (rs.next()) { return map(rs); }            // 找到 → 映射成 User
        }
    } catch (SQLException e) { e.printStackTrace(); }
    return null;                                           // 没找到 → null
}
```

**标准七步套路**:SQL(`?` 占位)→ 拿连接(`try-with-resources` 自动关)→ prepare → 填参(`setString`)→ 执行(`executeQuery`)→ 处理结果(`rs.next()` + `map`)→ 异常兜底。

> **防注入**:用户在手机号框输入 `' OR 1=1 --`,PreparedStatement 把它当**普通字符串**填进 `?`(自动转义),不会拼进 SQL → 攻不破。

### 9.2 `register(user)` —— 查重 + INSERT

> 📍 [`register()` L48–64](../../src/main/java/com/game/dao/UserDao.java#L48)

```java
if (findByPhone(user.getPhone())) {     // 先查重
    return false;                        // 手机号已存在
}
String sql = "INSERT INTO user(phone, password, nickname) VALUES (?, ?, ?)";
ps.setString(2, MD5Util.md5(user.getPassword()));   // 密码先 MD5 再存
return ps.executeUpdate() > 0;           // 影响行数 > 0 = 成功
```

**为什么先查重**:直接 INSERT 撞 UNIQUE 约束会报错,被 catch 吞成模糊的"注册失败"。先 `findByPhone` 能给**准确的"手机号已存在"**提示。

### 9.3 `changePassword(userId, oldPwd, newPwd)` —— 旧密码塞 WHERE ⭐

> 📍 [`changePassword()` L110–122](../../src/main/java/com/game/dao/UserDao.java#L110)

```java
String sql = "UPDATE user SET password = ? WHERE id = ? AND password = ?";
ps.setString(1, MD5Util.md5(newPwd));    // 新密码(要改成的)
ps.setInt(2, userId);
ps.setString(3, MD5Util.md5(oldPwd));    // 旧密码(塞进 WHERE 校验!)
return ps.executeUpdate() > 0;
```

**巧妙**:旧密码校验不在 Java 里 `if` 判断,而是**塞进 WHERE**——旧密码不对就匹配 0 行 → UPDATE 影响 0 行 → 返回 false。**一条 SQL 同时完成「校验旧密码 + 改新密码」**,不用先 SELECT 再 UPDATE(避免并发间隙)。

### 9.4 `deleteUser(userId)` —— 三表删除包事务 ⭐⭐

> 📍 [`deleteUser()` L152–219](../../src/main/java/com/game/dao/UserDao.java#L152)

```java
conn.setAutoCommit(false);                              // ① 开事务
// 先查 role:admin 不可删
if (!"user".equals(role)) { conn.rollback(); return false; }
// 清子表(战绩 + 重置申请)
DELETE FROM game_record WHERE user_id = ?
DELETE FROM password_reset_request WHERE user_id = ?
// 再删 user(admin 兜底:WHERE id=? AND role <> 'admin')
deleted = executeUpdate() > 0;
// ⭐ 只有真正删掉才提交,否则回滚
if (deleted) conn.commit(); else conn.rollback();
```

**为什么要事务**:三步删除(战绩 → 申请 → 用户),如果第三步失败、前两步已提交 → **孤儿数据**(战绩/申请在但用户没了)。事务保证**要么全删、要么一点不动**。

**`if(deleted) commit() else rollback()`**:不是无条件 commit——如果第三步因并发(role 被改成 admin)影响 0 行,必须 rollback 前两步(不留孤儿)。详见 [审计修复②](../v1.0_审计修复/扩展_审计修复_数据与玩法.md)。

### 9.5 `map(rs)` —— ResultSet → User

> 📍 [`map()` L222–234](../../src/main/java/com/game/dao/UserDao.java#L222)

```java
private User map(ResultSet rs) throws SQLException {
    User user = new User();
    user.setId(rs.getInt("id"));
    user.setPhone(rs.getString("phone"));
    ...
    return user;
}
```

把数据库一**行**映射成一个 Java **对象**。这是 DAO 层的标配——**ResultSet 是数据库视角(行+列),User 是 Java 视图(对象+字段),map 负责翻译**。

```java
           + "g.difficulty, g.record_time, u.nickname "                // JOIN 带出昵称
```

### 11.1 `requestReset(phone)` —— 三步:查 userId → 查重 → 插入

```java
SELECT id FROM user WHERE phone = ? → userId
```

### 11.2 `approve(requestId, userId)` —— 包事务:重置密码 + 改状态 ⭐

> 📍 [`approve()` L120–161](../../src/main/java/com/game/dao/ResetRequestDao.java#L120)

```java
conn.setAutoCommit(false);                                       // ① 开事务
// ② 重置密码(仅普通用户;admin 不可重置 → role='user' 兜底)
boolean reset = UPDATE user SET password=MD5('123456') WHERE id=? AND role='user' > 0;
// ③ 按结果置申请状态

UPDATE password_reset_request SET status=?, handle_time=NOW() WHERE id=?;

```

**为什么要事务**:两步写(改密码 + 改申请状态),要保证**要么都成、要么都废**——不能"密码改了但申请状态没更新"(否则同一条申请被重复审核)。

## 十二、DAO 层共性回顾

| 模式 | 体现 | 在哪 |
|---|---|---|
| **PreparedStatement 防注入** | 所有 SQL 用 `?` 占位 + `setString` 填值 | 每个方法 |
| **try-with-resources 自动关** | `try (Connection conn = ...; PreparedStatement ps = ...)` | 每个方法 |
| **map(rs) 结果集→对象** | ResultSet 一行 → Java 对象(User/GameRecord/PasswordResetRequest) | 每个 DAO |
| **事务(多步写)** | `setAutoCommit(false)` → 多步 → `commit/rollback` | deleteUser / approve |
| **JOIN 跨表取数据** | JOIN user 带出 nickname/phone | topN / listPending |

## 十三、小自测(dao 层)

1. **DBUtil 的 `static {}` 块什么时候执行?执行几次?**
2. **`login` 方法里,如果用户在手机号框输入 `' OR 1=1 --`,会登录成功吗?为什么?**
3. **`changePassword` 的旧密码校验为什么放 WHERE 而不是 Java 里 if 判断?**
4. **`deleteUser` 为什么要包事务?`if(deleted) commit() else rollback()` 改成无条件 `commit()` 会怎样?**
5. **`topN` 为什么要 JOIN user?不 JOIN 会怎样?**
6. **`mine` 和 `topN` 的 SQL 有什么区别?(提示:过滤条件 + 排序)**
7. **`approve` 为什么要包事务?为什么 `WHERE role='user'`?**

<details><summary>答案</summary>

1. **类第一次被用到时执行一次**(JVM 保证)。之后再调 getConnection 不重读配置——URL/USER/PASSWORD 是 static final 常量。
2. **不会**。PreparedStatement 把 `' OR 1=1 --` 当**普通字符串**填进 `?`(自动转义),SQL 变成 `WHERE phone = '\' OR 1=1 --' AND password = ?` → 查不到 → null → 登录失败。
3. 一条 SQL **原子**完成「校验旧密码 + 改新密码」,避免「先 SELECT 后 UPDATE」的并发间隙(SELECT 时旧密码对、UPDATE 前被人改了)。少一次 DB 往返。
4. 三步删除(战绩→申请→用户)要保证**要么全删要么一点不动**。无条件 commit:如果第三步(user)因并发影响 0 行(role 被改成 admin),前两步已提交 → **孤儿数据**(战绩/申请在但用户没了)。
5. 排行榜要显示**昵称**,但战绩表只有 `user_id`。JOIN user 把昵称拼过来。不 JOIN → 只有 user_id,界面显示数字 ID 不友好。
6. `topN`:`WHERE difficulty=? ORDER BY score DESC`(按难度 + 分数);`mine`:`WHERE user_id=? ORDER BY record_time DESC`(按用户 + 时间)。一个全局榜、一个个人历史。
7. 两步写(改密码 + 改申请状态)要保证一致——不能"密码改了但状态没更新"(会被重复审核)。`WHERE role='user'` = admin 账号不可被重置(影响 0 行 → reset=false → 申请置 rejected)。

</details>
