## §1. 一分钟说清这个项目（答辩开场陈述）

> 答辩第一句话最关键。背熟下面这段，开场就不会慌。

> 「这是一个**基于 Java Swing 的 2D 射击小游戏**（打僵尸）。玩家登录后用鼠标瞄准、左键射击，消灭从屏幕四边涌来的僵尸得分，血量归零结算并存档到 MySQL。
> 技术栈是 **Java（Swing + Java 2D）+ JDBC + MySQL 8**，Maven 管理依赖。
> 代码按 **UI / 游戏 / 数据 三层**组织，体现了**面向对象**——玩家、僵尸、子弹都实现一个 `GameObject` 接口，由游戏面板统一驱动。
> 除了核心玩法，我还做了**用户管理与账号安全**（管理员审核忘记密码、RBAC 角色）、**难度选择与分难度排行榜**、以及**纯代码合成的音效**。
> 整个项目经历了 8 个开发阶段，配套完整的需求 / 设计 / 数据库文档。」

**三个关键词记住**：① 三层架构 ② 面向对象（接口+多态）③ JDBC + MySQL 持久化。

## §3. 系统架构（代码是怎么组织的）

### 3.1 三层架构

```
┌─────────────────────────────────────────────┐
│   UI 层  (com.game.ui + GameApp)  Swing 窗体   │
│   LoginFrame / RegisterFrame / MainFrame      │
│   GameWindow / LeaderboardFrame               │
│   AdminFrame / ChangePasswordDialog           │
└──────────┬──────────────────────┬────────────┘
           │ 调用                  │ 持有/驱动
           ▼                      ▼
┌─────────────────────┐  ┌──────────────────────┐
│  游戏层 (com.game.game)│  │  数据层 (com.game.dao) │
│  GamePanel(渲染+事件) │  │  DBUtil(连接)          │
│  GameController(大脑) │  │  UserDao / RecordDao  │
│  Player/Zombie/Bullet │  │  ResetRequestDao      │
└─────────────────────┘  └──────────┬───────────┘
                                    │ JDBC
                                    ▼
                         ┌────────────────────┐
                         │  MySQL: game_db     │
                         │  user / game_record │
                         │  / password_reset_  │
                         │    request          │
                         └────────────────────┘
```

**原则**：层间**单向依赖**（UI→游戏→数据→DB），下层不知道上层存在；每个类**单一职责**。

### 3.3 关键类间关系（背熟，画类图用）

| 关系 | 体现的 OOP 思想 |
|---|---|
| `Player / Zombie / Bullet` 都 **实现** `GameObject` 接口 | **多态**：GamePanel 能用统一循环调 `update()/draw()`，加新对象不用改面板 |
| `GamePanel` **聚合** Player/Zombie/Bullet 列表 | 整体与部分（部分可独立存在） |
| `GamePanel` ⟷ `GameController` 互引 | 面板管渲染+事件，控制器管逻辑，**关注点分离** |
| `GameController` **依赖** `RecordDao` | 游戏结束时存档，**层间单向依赖**（游戏→数据） |
| `User 1 — N GameRecord` | 数据库**一对多**，外键 user_id |
| `GameWindow` **组合** `GamePanel` | 强生命周期（窗关则面板亡） |

## §4. 必须读懂的核心代码（精读路线）

> 「看懂代码」是这次准备的核心。按下面顺序读，每读完一个文件，确保能答出「读完要能说清」里的点。**不要跳读**，这是答辩底气。

### 第一站：程序怎么跑起来的（启动链）
1. [GameApp.java](../../src/main/java/com/game/GameApp.java) — 读完要能说清：为什么用 `SwingUtilities.invokeLater`？（EDT 线程安全，所有 Swing 操作必须在事件分派线程）；`UIStyle.initGlobalTheme()` 干嘛。
2. [util/UIStyle.java](../../src/main/java/com/game/util/UIStyle.java) — 暗色主题怎么全局生效（`UIManager.put` 改 LookAndFeel 默认值）。

### 第二站：数据怎么存的（数据层，DB 课核心）
3. [dao/DBUtil.java](../../src/main/java/com/game/dao/DBUtil.java) — **重点**：怎么读 `db.properties` 拿连接？`DriverManager.getConnection`？为什么 `close()` 要按 ResultSet→Statement→Connection 倒序关、且每步单独 try？（防一个失败导致后面资源泄漏）
4. [util/MD5Util.java](../../src/main/java/com/game/util/MD5Util.java) — `MessageDigest.getInstance("MD5")`，为什么存哈希不存明文（安全），MD5 的局限见 §7。
5. [dao/UserDao.java](../../src/main/java/com/game/dao/UserDao.java) — **重中之重**：① `register`/`login` 为什么用 `PreparedStatement`？（防 SQL 注入）② `login` 的 SQL 是 `WHERE phone=? AND password=?`（把 MD5 密码也放进 WHERE）；③ `deleteUser` 的**事务**怎么写的（setAutoCommit(false)→三条 DELETE→`if(deleted) commit() else rollback()`→finally 还原+close），为什么 commit 要带条件（防 0 行删除造孤儿）；④ `changePassword` 用 `WHERE id=? AND password=?` 顺带校验旧密码。
6. [dao/RecordDao.java](../../src/main/java/com/game/dao/RecordDao.java) — `saveRecord`（INSERT 5 列含 difficulty）、`topN(n, difficulty)`（`WHERE difficulty=? ... LIMIT ?`）、`mine(userId, difficulty)`；为什么排行榜查询走索引 `idx_diff_score`。
7. [dao/ResetRequestDao.java](../../src/main/java/com/game/dao/ResetRequestDao.java) — `requestReset`（先 `hasPending` 查重再插）、`approve`（重置成 `MD5('123456')` + 置 approved + 记 handle_time，**包事务**）、`listPending`（JOIN user 拿 phone/nickname 显示）。
8. [model/User.java](../../src/main/java/com/game/model/User.java) / [GameRecord.java](../../src/main/java/com/game/model/GameRecord.java) / [PasswordResetRequest.java](../../src/main/java/com/game/model/PasswordResetRequest.java) — POJO，字段对应表列。

### 第三站：界面怎么搭的（UI 层）
9. [ui/LoginFrame.java](../../src/main/java/com/game/ui/LoginFrame.java) — 登录校验链；「忘记密码」入口；回车登录（键盘事件）；登录成功记录 currentUser 并 `new MainFrame`。
10. [ui/RegisterFrame.java](../../src/main/java/com/game/ui/RegisterFrame.java) — 输入校验：**手机号 11 位数字**、密码≥6、昵称 ≤ 50、手机号重复提示。
11. [ui/MainFrame.java](../../src/main/java/com/game/ui/MainFrame.java) — 主菜单；**admin 才显示「用户管理」按钮**（`if(role==admin)`）；「开始游戏」弹难度选择（简单/困难）；`showHelp` 的 7 条说明（要对得上代码）。
12. [ui/GameWindow.java](../../src/main/java/com/game/ui/GameWindow.java) — **重点**：① 用 `javax.swing.Timer` 驱动 GamePanel；② **点 X 的结算存档**：`DO_NOTHING_ON_CLOSE` + 手动 dispose + 确认框，为什么模态框**不会**自动停 Timer（要先 panel.stop()，取消再 start()）；③ `settled` 标志防重复存档。
13. [ui/LeaderboardFrame.java](../../src/main/java/com/game/ui/LeaderboardFrame.java) — 简单榜/困难榜、全局/我的记录四个组合；JTable + DefaultTableModel。
14. [ui/AdminFrame.java](../../src/main/java/com/game/ui/AdminFrame.java) — 双 JTable（用户表 + 待审表）；通过/拒绝/删除按钮。
15. [ui/ChangePasswordDialog.java](../../src/main/java/com/game/ui/ChangePasswordDialog.java) — 模态对话框，调 changePassword。

### 第四站：游戏核心（Java 课核心，最能体现 OOP）
16. [game/Difficulty.java](../../src/main/java/com/game/game/Difficulty.java) — **亮点**：把刷怪间隔/封顶/Brute 门槛/概率/伤害做成枚举字段。为什么不用 `static final` 常量？（参数要随难度变，常量是全局唯一的；放枚举里 = 每个难度一套独立参数，构造时注入）。这是**依赖注入/参数化**思想。
17. [game/GameObject.java](../../src/main/java/com/game/game/GameObject.java) — 接口，两个方法。为什么用接口而不是抽象父类？（Player/Zombie/Bullet 没有共享的实现代码，只需要统一的行为契约；接口更轻、支持多实现）。
18. [game/Player.java](../../src/main/java/com/game/game/Player.java) — 位置、hp=100、angle、radius、`SPEED` 常量、`takeDamage`、移动边界钳制。
19. [game/Zombie.java](../../src/main/java/com/game/game/Zombie.java) — **重点**：`Type` 枚举（NORMAL/BRUTE）→ 半径(18/26)、血量、分值(10/25)、速度、血液颜色都随类型变；`setTarget` + `update` 追踪玩家（朝向归一化移动）。
20. [game/Bullet.java](../../src/main/java/com/game/game/Bullet.java) — 位置 + (vx,vy) 速度向量；`isOffscreen` 出界判断。
21. [game/GameController.java](../../src/main/java/com/game/game/GameController.java) — **重中之重，整颗大脑**。逐方法读懂：
    - `update()`：一帧的完整步骤（移动→推进对象→推进特效→刷怪→碰撞→清理→结束判定）。**为什么 update() 里「碰撞只改状态、清理用 removeIf 单独做」**？（遍历时不能增删集合，否则 ConcurrentModificationException）。
    - `shoot()`：枪口位置 = 玩家中心 + 朝向×(半径+8)；子弹速度 = cos/sin × BULLET_SPEED。
    - `spawnZombie()`：**难度随时间递增**——`interval = max(minSpawn, spawnInterval - elapsedSec)`，越往后刷得越快；Brute 门槛 `elapsedSec >= bruteMinSec && 随机<bruteChance`；**生成安全距离**循环（防贴墙玩家被凭空撞）。
    - `handleCollisions()`：**圆形碰撞** = 圆心距 < 半径和；子弹×僵尸（一颗子弹只命中一只，break）、僵尸×玩家（扣血+僵尸消失，防反复扣）。
    - `getElapsedSec()`：基于 frameCounter 折算，**暂停不计入**（因为暂停时 update 不被调）。
22. [game/GamePanel.java](../../src/main/java/com/game/game/GamePanel.java) — **重点**：① `javax.swing.Timer`（约 16ms/帧≈60FPS）每 tick 调 `controller.update()` 再 `repaint()`；② 键盘事件（WASD/方向键）设 moveDir、鼠标事件设 aim + 左键 shoot；③ `paintComponent` 分层绘制（背景→世界对象+震屏偏移→粒子→浮动文字→HUD→受伤红屏叠加）；④ 暂停/恢复（timer.stop/start）。
23. [game/Particle.java](../../src/main/java/com/game/game/Particle.java) / [FloatingText.java](../../src/main/java/com/game/game/FloatingText.java) — 视觉特效，生命周期 + isDead 清理。

### 第五站：音效（独立亮点，老师可能问「你怎么生成声音的」）
24. [util/SoundUtil.java](../../src/main/java/com/game/util/SoundUtil.java) — **深讲见 §5.6**。重点：单 SourceDataLine 复用 + BlockingQueue + 守护线程；纯代码合成 PCM（44100Hz/16bit/mono）；小缓冲 + 起音/释音淡入淡出。
25. [util/GameSettings.java](../../src/main/java/com/game/util/GameSettings.java) — 音量/静音全局态。

## §5. 答辩亮点与技术深讲（老师会追问的）

> 挑你最有把握的 3~4 个重点讲。下面每条都给「是什么 / 为什么这么做 / 怎么实现 / 可能被问什么」。

### 5.1 三层架构 + 单一职责
- **是什么**：UI / 游戏 / 数据 三层，单向依赖；每类只干一件事。
- **为什么**：解耦——改数据库不影响游戏逻辑、改界面不影响数据。可维护、可测试（GameController 不依赖 Swing，能脱离界面单测，见 TestGameLogic）。
- **被问**：「为什么不让 GamePanel 直接写 SQL？」「你的层间依赖方向？」「Controller 为什么不放进 GamePanel？」→ 都答：**单一职责 + 解耦 + 可测试**。

### 5.2 GameObject 接口 + 多态 ⭐（最能体现 OOP，必讲）
- **是什么**：`Player/Zombie/Bullet` 都实现 `GameObject{update();draw(g);}`，GamePanel 用统一列表 + 统一循环驱动。
- **为什么**：加新对象（比如未来加「道具」「陷阱」）不用改面板的遍历代码——**对扩展开放、对修改关闭（开闭原则）**。这是教科书级的**多态**应用。
- **被问**：「接口和抽象类你怎么选的？」「多态在你项目里体现在哪？」→ 指这里。

### 5.3 游戏主循环（Timer + EDT）
- **是什么**：`javax.swing.Timer` 每 ~16ms 触发一次 → 调 `controller.update()` 推进一帧 → `repaint()` 重绘。
- **为什么用 swing.Timer 而不是 Thread**：Timer 的回调**自动在 EDT 上执行**，天然线程安全，不会和界面渲染抢；写起来简单。
- **被问**：「你的多线程在哪？」→ 守护消费线程（SoundUtil）、EDT（Timer/界面）。要诚实：游戏主循环是**单线程 EDT**，没做多线程游戏逻辑（见 §7 局限）。
- ⚠️ **坑点**（你在审计里踩过，能讲）：模态对话框**不会**自动停 Timer（它跑嵌套事件泵继续派发 tick）→ 暂停/确认前必须手动 `timer.stop()`。

### 5.4 圆形碰撞检测
- **是什么**：`Math.hypot(dx,dy) < 半径和` 判定两圆相交。
- **为什么**：圆形对象用距离判定最简单准确，比矩形 AABB 更贴合视觉。
- **被问**：「碰撞怎么做的？」「子弹打多个僵尸吗？」→ 一颗子弹 `break` 只命中一只；僵尸撞玩家后**消失**（防同只僵尸反复扣血）。

### 5.5 难度系统（enum 参数化 + 时间递增）⭐
- **是什么**：`Difficulty` 枚举把刷怪间隔/封顶/Brute 门槛/概率/伤害封装成字段；`GameController` 构造时注入难度。
- **两层设计**：① **档位**（简单/困难）= 不同参数集，注入式；② **档内递增** = `interval = max(minSpawn, spawnInterval - elapsedSec)`，同一档里越打越快，有上限不失控。
- **为什么不用 static final 常量**：常量是全局唯一，无法支持「简单一套、困难一套」；枚举字段 = 每档一套独立参数，干净。
- **被问**：「难度怎么实现的？」「僵尸越来越快怎么做到的？」→ 指这里。这是**策略/参数注入**思想的雏形。

### 5.6 音效纯代码合成 ⭐（差异化亮点，别人都用音频文件）
- **是什么**：不依赖任何 wav/mp3，用代码现场算 PCM 采样喂给 `SourceDataLine` 播。四种音：开火(600Hz正弦40ms)、命中(150Hz)、受伤(400→200Hz下滑)、结束(递降旋律)。
- **合成原理**：44100Hz/16bit/单声道；方波=signum(sin)、正弦=sin；叠加指数衰减包络 `exp(-decay·t)`；频率扫描用**相位累加**（`phase += 2π·freq·dt`）避免跳变咔哒声；振幅按音量缩放、short 拆小端序两字节。
- **播放模型**：全局**单条** SourceDataLine（懒加载复用，避免反复 open/close 耗尽线路）+ BlockingQueue + 守护消费线程（串行写、不并发冲突）。
- **你修过的两个坑**（很能讲，见[音效修复笔记](../v1.0_审计修复/扩展_音效修复_射击干脆与松手即停.md)）：
  - 「松手还在响」→ 根因是 `line.write()` 只在缓冲**满**时阻塞、默认大缓冲让音频囤积；修法：`open(format, 小缓冲≈50ms)` 把消费线程节流到实时。
  - 「连按乱响/爆音」→ 根因是每个音只有开头淡入没结尾淡出、衔接处振幅突变；修法：加 5ms 结尾淡出（release），与起音对称。
- **被问**：「你的音效文件在哪？」「多线程怎么保证不冲突？」「为什么松开还在响你怎么解决的？」→ 全在这。

### 5.7 数据库设计 + JDBC 安全
- **3 表 + 外键一对多**：user 1—N game_record、user 1—N password_reset_request。
- **PreparedStatement 防注入**：所有 SQL 参数化（`?`），不拼字符串。被问「怎么防 SQL 注入」→ 指这里 + 举例 `login` 的 `WHERE phone=? AND password=?`。
- **索引**：`idx_diff_score(difficulty, score)` 加速分难度排行榜、`idx_user`/`idx_score` 等。被问「排行榜查询为什么快」→ 走索引 + `LIMIT`。
- **事务**：deleteUser 三表删除、ResetRequestDao.approve 重置+改状态——`setAutoCommit(false)` 手动控提交点，`commit/rollback`，finally 还原+close。被问「事务用来保证什么」→ **原子性 all-or-nothing**（删用户要么全删要么不动；commit 要带条件 `if(deleted)`）。
- **密码**：MD5 哈希存（见 §7 局限）。

### 5.8 RBAC 账号安全
- **是什么**：`user.role`（admin/user）；admin 多「用户管理」入口（MainFrame 按 role 显示按钮）；admin 可审核重置申请、删用户；**admin 账号不可删、不可被重置**（DAO 层先查 role 拦截）。
- **忘记密码流程**：用户提交→pending→admin 通过→密码重置成 123456→用户登录后自改。同用户有 pending 时不可重复提交（hasPending 查重）。
- **被问**：「权限怎么控制的？」「为什么 admin 不能被删？」「忘记密码为什么不直接发邮件？」（见 §7：课设场景用人工审核替代邮件）。

### 5.9 视觉打磨
- 粒子（枪口火花/命中火花/僵尸爆血）、浮动得分文字（+10/+25）、震屏（线性衰减）、受伤红屏、精灵图角色。
- 体现**用户体验**和**Graphics2D 绘制**能力。被问「画面效果怎么做的」→ 指 GameController 里的 spawn* 方法和 GamePanel 分层绘制。

### 5.10 健壮性
- **异常吞掉不崩**：音效/数据库异常 catch 后给友好提示或静默，游戏照跑。
- **输入校验靠前**：注册界面就拦长度/重复，不让脏数据捅到 DB 报无关错误。
- **生成安全距离**：僵尸不在玩家身边凭空生成（防不公平）。

18. **怎么避免连射时声音冲突/卡顿？** → 单线路复用 + 队列 + 守护线程串行；小缓冲防囤积；淡入淡出防爆音。
19. **图片怎么处理的？** → 精灵图抠图去水印（见[Java图片处理笔记](../Java图片处理(抠图与精灵图).md)）。

### 安全 / 账号

21. **admin 账号怎么保护？** → 不可删、不可被重置（DAO 查 role 拦截）。

## §7. 诚实地谈「已知局限」（加分项）

> 老师很看重你**知不知道自己的不足**。下面 7 条是课设阶段的**有意权衡**（不是 bug），坦诚说出来 + 给改进方向 = 大加分。详见 [../../docs/已知问题与后续优化.md](../../docs/已知问题与后续优化.md)。

| # | 局限 | 答辩怎么说（坦诚 + 有方向） |
|---|---|---|
| 1 | **密码 MD5 无盐** | 「MD5 已不安全，生产应换 BCrypt/PBKDF2 加随机盐，并在登录时对老哈希惰性重哈希。课设为简化沿用 MD5，这是我已知的待改进点。」（**这条最可能被问，准备好**） |
| 2 | 重置成硬编码 123456 | 「应改成随机一次性密码/邮件 token + 首登强制改。课设用人工审核替代了邮件。」 |
| 3 | DAO 在 EDT 同步调用 | 「数据库慢时界面会卡。应用 SwingWorker 挪出 EDT + URL 加超时。本地 MySQL 几乎不卡。」 |
| 4 | Test 类在 main 里 | 「应挪到 src/test/java + JUnit。当前是手动测试类。」 |
| 5 | 忘记密码可枚举手机号 | 「应统一回『若该用户存在，申请已提交』。本地无攻击者。」 |
| 6 | requestReset 跨连接竞态 | 「hasPending 和插入不在同一事务，理论有 TOCTOU。应并同一连接 + 唯一约束。」 |
| 7 | AdminFrame 按钮分组 | 「功能没错，体验上按钮可重新分组。」 |

> **话术要点**：先承认「这是已知权衡」，再说「为什么课设阶段可接受」，最后给「生产级怎么修」。**不要装作没这个问题**——老师一问就露馅；主动说出来反而显专业。

## §8. 演示流程与讲解脚本

### 8.1 实机演示路径（run.bat 启动后，按这个顺序演，覆盖全部功能）
1. **启动** → 登录界面（`run.bat`，先确认 MySQL 起着、db.properties 配好）。
2. **注册** → 新注册一个号（演示输入校验：故意输非 11 位、密码<6、重复手机号看提示）。
3. **登录普通用户** → 主菜单（注意：没有「用户管理」按钮，因为不是 admin）。
4. **开始游戏 → 选「困难」** → 进游戏（演：WASD 移动、鼠标瞄准、左键射击、连射、被打到扣血+震屏+红屏、击杀飘 +10/+25、击杀壮汉）。
5. **打一会儿故意死** → 结算面板（分数/击杀/用时）→ 看排行榜 / 再来一局。
6. **点 X 测试**（再开一局中途点窗口 X）→ 弹确认 → 选「是」→ 验证也存档了。
7. **排行榜** → 切「简单榜/困难榜」「全局/我的记录」四个组合。
8. **退出登录 → 用 admin 登录**（admin/admin 默认 123456）→ 主菜单出现「用户管理」。
9. **用户管理** → 看用户表；点「忘记密码」用普通号提交一个申请 → admin 刷新待审表 → 通过 → 提示重置成 123456。
10. **修改密码** → 用重置后的号 + 123456 登录 → 改密码。
11. **设置** → 切音效开关/音量，回游戏验证。
