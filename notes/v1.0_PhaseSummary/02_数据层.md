## 1. 开发流程（这阶段怎么走过来的）

```
探查环境 → 选 Maven → 写 pom/db.properties → 写代码(MD5→DBUtil→实体→DAO→TestDao)
       → javac 编译验证 → 验证 MD5 对齐 → 建库(schema.sql) → 下载驱动跑 TestDao → 更新计划书
```

1. **探查环境**：发现还没有 Java 工程、JDK 是 25、缺驱动 jar。
2. **选 Maven**（问过你）：pom.xml 自动管理驱动，不用手动下 jar。
3. **写工程骨架**：`pom.xml` + `db.properties`。
4. **按依赖顺序写代码**：`MD5Util → DBUtil → User/GameRecord → UserDao/RecordDao → TestDao`。
5. **javac 编译验证**：全部 0 报错（驱动 jar 运行时才需要，编译不需要）。
6. **验证 MD5**：确认 `MD5("123456")` = schema.sql 里的值，密码会对得上。
7. **建库**：用 `mysql` 跑 schema.sql，建好 game_db + 3 用户 + 5 战绩。
8. **跑 TestDao**：下载驱动 jar → 编译 → 运行，全流程通过。

> **复盘要点**：依赖关系决定书写顺序——被依赖的先写（MD5Util 最先，因为它最独立且被 UserDao 用），这样每写完一个都能编译通过，**不会攒一堆错**。

## 2. 关键设计决策（为什么这么定）

| 决策 | 选择 | 为什么 |
|---|---|---|
| 依赖管理 | Maven | pom.xml 一行声明驱动，自动下载；不用手动下 jar、配 classpath |
| 配置外置 | `db.properties` 放 classpath | 改密码/换库不用动代码、不用重新编译 |
| 关资源 | `try-with-resources` | 自动关闭连接/语句/结果集，**防连接泄漏**（忘了关连接会慢慢耗光） |
| SQL | 全部 `PreparedStatement` | 防止 SQL 注入（用户名里填 `' OR 1=1` 也攻不破） |
| 密码 | MD5 存、MD5 比 | 不存明文；注册写入前、登录比对前都先 MD5 |
| 架构 | DAO 模式 | 把"数据库怎么存"封在 DAO 里，业务/界面不直接碰 SQL |

## 3. 核心知识点（复盘重点，建议逐条吃透）

### 3.1 JDBC 六步法（增删改查的套路）
```
1. 获取连接 Connection（DBUtil.getConnection）
2. 创建预编译语句 PreparedStatement（带 SQL，? 是占位符）
3. 填参数 ps.setXxx(下标, 值)
4. 执行：查询 executeQuery()→ResultSet；增删改 executeUpdate()→影响行数
5. 处理结果（ResultSet 遍历 / 映射成对象）
6. 关资源（try-with-resources 自动关）
```
记住这六步，所有数据库操作都是它的变体。

### 3.2 PreparedStatement vs Statement
- `Statement`：SQL 字符串拼接 → **有 SQL 注入风险**，且每次都要重新解析。
- `PreparedStatement`：用 `?` 占位 + `setXxx` 填值 → **防注入 + 可预编译更快**。
- **结论：永远用 PreparedStatement**。

### 3.3 try-with-resources
```java
try (Connection conn = DBUtil.getConnection();
     PreparedStatement ps = conn.prepareStatement(sql)) {
    // 用 conn 和 ps
}  // ← 出括号自动 close，即使抛异常也会关
```
凡是实现了 `AutoCloseable` 的（Connection/Statement/ResultSet/InputStream）都能放进去，**告别手写 finally 关闭**。

### 3.4 配置外置（Properties + classpath）
- `db.properties` 放 `src/main/resources`，编译后自动到 classpath 根。
- 用 `类.class.getClassLoader().getResourceAsStream("db.properties")` 读取。
- 好处：换环境（家/学校）只改文本文件，不动代码。

### 3.5 DAO 模式
- 每张表一个 DAO（`UserDao` 管 user、`RecordDao` 管 game_record）。
- DAO 里只写 SQL 和对象映射，**不掺业务逻辑**。
- 上层（界面/游戏）只调 DAO 方法（`login()`、`topN()`），不知道 SQL 长啥样 → **解耦**。

### 3.6 ResultSet → 对象映射
```java
while (rs.next()) {
    User u = new User();
    u.setId(rs.getInt("id"));
    u.setUsername(rs.getString("username"));
    // ...
}
```
这就是 ORM（对象关系映射）最原始的样子，MyBatis/Hibernate 本质也在干这个。

### 3.7 Maven 三连（pom.xml）
- **坐标**：groupId + artifactId + version，唯一定位一个库。
- **依赖 `<dependency>`**：声明"我需要某个库"，Maven 自动从中央仓库下载。
- **本地仓库**：下载的 jar 存在 `~/.m2/repository`，下次直接用。

## 4. 踩坑 / 注意点（记下来）

| 坑 | 现象/原因 | 解法 |
|---|---|---|
| **MySQL 8 时区** | 不带时区直接报错 | URL 加 `serverTimezone=Asia/Shanghai` |
| **MySQL 8 认证** | 连接报 public key 错 | URL 加 `useSSL=false&allowPublicKeyRetrieval=true` |
| **中文乱码（数据）** | 存进去的中文变问号 | URL 加 `characterEncoding=utf8` + 库用 utf8mb4；mysql 客户端加 `--default-character-set=utf8mb4` |
| **MD5 不一致** | 注册能成功但登录失败 | MD5 实现要对齐数据库（这次专门验证过 `123456`→`e10adc…883e`） |
| **驱动 jar 的时机** | 编译报错找不到类？不会 | java.sql 在 JDK 里；驱动 jar **编译不需要、运行需要**（DriverManager 运行时加载） |
| **忘了关连接** | 跑几次后连不上库 | 用 try-with-resources 自动关 |
| **控制台中文乱码** | 打印的中文变方块 | 不影响数据正确性，IDEA 控制台正常显示 |
