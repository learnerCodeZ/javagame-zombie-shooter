## 2. 关键设计决策

| 决策 | 选择 | 为什么 |
|---|---|---|
| 窗口释放 | 跳转时用 `dispose()` 不用 `setVisible(false)` | JFrame 是重量级窗口，只隐藏不会被垃圾回收，会**泄漏内存** |
| 数据库异常 | UI 层 `try/catch` 兜底 | 连不上库时弹"数据库连接失败"友好提示，不让界面崩 |
| 线程 | `SwingUtilities.invokeLater` 创建窗口 | Swing **不是线程安全的**，UI 操作必须在 EDT（事件分派线程）上做 |
| 外观 | `setLookAndFeel(系统外观)` | 让窗口跟着系统风格走，比默认金属外观好看 |
| 占位 | 开始游戏/排行榜先弹提示 | 这俩是阶段④⑤的活，先把骨架搭好 |

## 3. 核心知识点（Swing 基础，复盘重点）

### 3.1 Swing 组件层级
```
JFrame（顶层窗口）
 └─ JPanel（容器/面板）
     ├─ JLabel（文字）/ JTextField（输入框）/ JPasswordField（密码框）
     └─ JButton（按钮）
```
一层套一层，`JFrame` 是最外层窗户，`JPanel` 是窗户里的板子，组件钉在板子上。

### 3.2 布局管理器（决定组件怎么摆）
本项目用全了四种，很典型：
| 布局 | 长啥样 | 本项目用在哪 |
|---|---|---|
| `BorderLayout` | 东南西北中 5 个区 | 窗口：标题在北、表单在中、按钮在南 |
| `GridLayout` | 规整网格 | 表单：标签+输入框 两列对齐 |
| `FlowLayout` | 从左到右排 | 底部按钮区 |
| `BoxLayout`(Y) | 纵向堆叠 | 主菜单 5 个大按钮 |

### 3.3 事件驱动编程
Swing 是"**你点什么，它回调什么**"：
```java
loginButton.addActionListener(e -> doLogin());  // 点登录 → 执行 doLogin()
```
`e -> ...` 是 lambda，相当于"点击时要做的事"。这是 GUI 编程的核心心智模型。

### 3.4 EDT 与 SwingUtilities.invokeLater
- Swing **不是线程安全的**：所有 UI 创建/修改必须在 **EDT（事件分派线程）** 上。
- `SwingUtilities.invokeLater(() -> ...)` 把任务排到 EDT 执行。
- 所以 `main` 里打开窗口要包一层 `invokeLater`。

### 3.5 窗口生命周期（这次踩坑的点）
| 操作 | 效果 |
|---|---|
| `setVisible(false)` | **只隐藏**，窗口对象还在内存，不会被回收 |
| `dispose()` | **真正释放**窗口资源 |
| `EXIT_ON_CLOSE` | 关窗 = 整个程序退出 |
| `DISPOSE_ON_CLOSE` | 关窗 = 只关本窗，程序继续 |

> 登录窗跳注册窗用 `dispose()`（不是 `setVisible(false)`），就是因为注册窗回来时会 new 新登录窗，老的不 dispose 就泄漏了。

### 3.6 JOptionPane（快捷弹窗）
```java
JOptionPane.showMessageDialog(this, "用户名或密码错误", "登录失败", JOptionPane.ERROR_MESSAGE);
```
一行搞定提示框，四种图标：`WARNING_MESSAGE` / `ERROR_MESSAGE` / `INFORMATION_MESSAGE`。

## 4. 踩坑 / 注意点

| 坑 | 现象 | 解法 |
|---|---|---|
| **窗口泄漏** | 反复 登录→注册→返回，隐藏的登录窗越攒越多 | 跳转用 `dispose()`，不用 `setVisible(false)` |
| **DB 异常未捕获** | MySQL 没启动时点登录，界面崩/报一堆红字 | UI 层 `try/catch` 兜底，弹"数据库连接失败" |
| **EDT 违规** | 在 main 线程直接 new 窗口 | 用 `SwingUtilities.invokeLater` |
| **回车登录** | 用户期望按回车就能登录 | `getRootPane().setDefaultButton(登录按钮)` |
