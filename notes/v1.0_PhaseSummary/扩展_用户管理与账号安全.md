## 2. 核心知识点（复盘重点）⭐

### 2.1 角色权限（RBAC 雏形）
- `user` 表加 `role` 列（`admin`/`user`）。
- 界面按角色显隐：`MainFrame` 里 `"admin".equals(user.getRole())` 才显示"用户管理"按钮。
- 这就是**基于角色的访问控制**的最简形态——一个字段决定能看/能做什么。

### 2.2 密码重置：申请-审核两步流程
不是"点忘记密码直接改"，而是**两步走**（更真实、更安全）：
1. 用户点"忘记密码" → 输用户名 → 写一条 `pending` 申请。
2. 管理员登录 → 在"用户管理"看到 pending 申请 → 点"通过" → 密码重置为 `123456` → 用户用 123456 登录后自己改。
> 这种"申请→审批"模式在企业系统里很常见（请假、报销、开户…）。

### 2.3 数据库事务（approve 的关键）⭐
`approve` 要做两件事：①重置用户密码 ②把申请标 approved。必须**要么都成功、要么都不做**。用事务保证：
```java
conn.setAutoCommit(false);   // 开启事务
// ① UPDATE user SET password=... 
// ② UPDATE password_reset_request SET status='approved'...
conn.commit();               // 都成功才提交
// 出异常 → conn.rollback(); 回滚
```
> 事务（ACID）是数据库的核心：保证多步操作的**原子性**。这是答辩加分点。

### 2.4 admin 保护（超级账号不可动）
多处加守卫，admin 账号既**不可删**也**不可被重置**：
- `deleteUser`：先查 role，admin 直接拒绝（连它的数据都不碰）。
- `approve`：`UPDATE ... WHERE role='user'`，admin 的密码 UPDATE 影响 0 行 → 申请转 rejected。

### 2.5 级联删除（外键约束的处理）
表之间有外键（`game_record`/`password_reset_request` 都引用 `user`）。直接删 user 会因外键约束失败。解法：**先删依赖再删主表**（手动级联）：
```
DELETE game_record WHERE user_id=? → DELETE password_reset_request WHERE user_id=? → DELETE user
```
> 外键是 RESTRICT（默认），不像 CASCADE 自动连带删，所以要手动清依赖。

### 2.6 用 UPDATE...WHERE 做原子校验（changePassword）
改密码要"旧密码对才改"。巧妙写法：
```sql
UPDATE user SET password=MD5(新) WHERE id=? AND password=MD5(旧)
```
一条 SQL 同时**校验旧密码 + 改新密码**，按影响行数判断（0 行=旧密码错）。比"先查后改"更简洁、避免竞态。

### 2.7 JOIN 取关联显示字段
`listPending` 要显示"谁申请的"，但 reset 表只有 user_id。用 JOIN 把 user 表的 username/nickname 拼过来：
```sql
SELECT r.*, u.username, u.nickname FROM password_reset_request r JOIN user u ON r.user_id=u.id ...
```

## 3. 踩坑 / 注意点

| 坑 | 解法 |
|---|---|
| 删 admin 会先清掉它的数据再被拦 | deleteUser 先查 role 预检，admin 直接拒绝、不碰任何记录 |
| 外键约束让删 user 失败 | 先删 game_record/reset 依赖，再删 user |
| approve 两步要原子 | 用事务（setAutoCommit/commit/rollback） |
| 同一用户重复提交申请 | requestReset 先 hasPending 查重 |
| DAO 把异常吞成 boolean | UI 区分不了"业务假"vs"DB错"（低，桌面单用户可接受） |
| requestReset 用两条连接查重+插入 | 理论 TOCTOU 竞态（低，单用户桌面可忽略；要严谨可合并到一个事务） |
| listAllUsers 取了 password 哈希 | 凭据卫生（低，本地应用；可改成不 SELECT password） |

## 4. 怎么用（你来验证）

`run.bat` → 登录：
