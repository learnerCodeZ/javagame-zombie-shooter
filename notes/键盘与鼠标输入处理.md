# 键盘与鼠标输入处理

| 项 | 内容 |
|---|---|
| 问题 | 游戏里 WASD 移动、鼠标瞄准、左键射击——这些输入是怎么接入代码的? |
| 核心 | 全在 [`GamePanel.java`](../src/main/java/com/game/game/GamePanel.java) 里,用 Swing 的「监听器(listener)」模式 |

---

## 一、核心概念:事件驱动 + 监听器

Swing 是**事件驱动**的:你不主动"等"按键,而是**注册一个监听器**——告诉 Swing「事件来了叫我」,然后该干嘛干嘛。当事件发生(按了键 / 动了鼠标),Swing 在 **EDT**(事件分派线程)上自动调你的回调方法。

> 这就是**观察者模式(Observer)**:你(监听器)订阅事件源(Swing 组件),事件来了 Swing 通知你。

---

## 二、键盘:WASD 移动

### 2.1 监听按键按下 / 松开

> 📍 [`GamePanel.java` · `pressedKeys` 字段 第 50 行](../src/main/java/com/game/game/GamePanel.java#L50) · [`KeyAdapter` 第 114–124 行](../src/main/java/com/game/game/GamePanel.java#L114)

```java
// GamePanel 第 50 行:
private final Set<Integer> pressedKeys = new HashSet<>();   // ← 记录"当前按住了哪些键"

// GamePanel 第 114–124 行:
addKeyListener(new KeyAdapter() {
    @Override
    public void keyPressed(KeyEvent e) {        // L116
        pressedKeys.add(e.getKeyCode());          // 按下 → 把键码加进 Set
    }
    @Override
    public void keyReleased(KeyEvent e) {        // L121
        pressedKeys.remove(e.getKeyCode());        // 松开 → 从 Set 移除
    }
});
```

- `KeyAdapter` = Swing 的键盘监听器基类(空实现了所有方法,你只重写需要的)。
- `KeyEvent e` = 按键事件,`e.getKeyCode()` = 按了哪个键(如 `KeyEvent.VK_W` = 87)。
- **为什么用 `Set` 不用单个变量**:如果只存一个键,按住 W 再按 D 就丢了 W。Set **同时记多个键**,所以 W+D 同按能斜走。

### 2.2 每帧从 Set 合成移动方向(轮询)

> 📍 [`GamePanel.java` · `moveDirX()` 第 183 行](../src/main/java/com/game/game/GamePanel.java#L183) · [`moveDirY()` 第 195 行](../src/main/java/com/game/game/GamePanel.java#L195) · [`tick()` 第 138–139 行](../src/main/java/com/game/game/GamePanel.java#L138)

```java
// tick() 里每帧调一次(L139):
controller.setMoveDir(moveDirX(), moveDirY());

// moveDirX()(L183):
private int moveDirX() {
    int dx = 0;
    if (pressedKeys.contains(VK_D) || pressedKeys.contains(VK_RIGHT)) dx++;  // 右 +1
    if (pressedKeys.contains(VK_A) || pressedKeys.contains(VK_LEFT))  dx--;  // 左 -1
    return dx;   // -1=左, 0=不动, +1=右
}
// moveDirY() 同理(L195,W/UP=-1, S/DOWN=+1)
```

**这是「轮询(polling)」**:不是按键时才动,而是**每帧检查 Set 里有哪些键**,合成方向喂给 controller。按住 W = 每帧都有"上"方向 = 持续移动。

> **为什么用轮询、不在 keyPressed 里直接移动**:`keyPressed` 按住只触发**一次**(不是每帧都触发),直接在里面 `player.y -= speed` 只动一下。用 Set + tick 轮询,按住 = 持续动。这是**连续移动**的标准做法。

### 2.3 失焦清空(防卡键)

> 📍 [`GamePanel.java` · `focusLost` 第 126–131 行](../src/main/java/com/game/game/GamePanel.java#L126)

```java
// GamePanel 第 126–131 行:
addFocusListener(new FocusAdapter() {
    @Override
    public void focusLost(FocusEvent e) {        // L128
        pressedKeys.clear();                      // 窗口失焦 → 清空按住的键
    }
});
```

**为什么**:按住 W 然后 Alt+Tab 切走,窗口失焦——`keyReleased` 可能不触发(Swing 收不到松键事件)。如果不 clear,W 永远留在 Set 里 → 回来后角色一直往上跑(卡键)。失焦清空防这个。

### 2.4 让键盘生效的前提:setFocusable + requestFocus

> 📍 [`GamePanel.java` · `setFocusable` 第 64 行](../src/main/java/com/game/game/GamePanel.java#L64) · [`requestFocusInWindow` 第 151 行](../src/main/java/com/game/game/GamePanel.java#L151)

```java
setFocusable(true);                        // L64,JPanel 默认不可聚焦,要打开
// start() 里(L151):
SwingUtilities.invokeLater(this::requestFocusInWindow);   // 请求焦点
```

JPanel 默认 `focusable = false`,不设的话键盘事件**根本收不到**。`requestFocusInWindow` 让画布拿到焦点(才能接收按键)。

## 三、鼠标:瞄准 + 射击

### 3.1 鼠标移动 → 瞄准(MouseMotionListener)

> 📍 [`GamePanel.java` · `mouseMoved` 第 80–86 行](../src/main/java/com/game/game/GamePanel.java#L80) · [`mouseDragged` 第 89–93 行](../src/main/java/com/game/game/GamePanel.java#L89)

```java
// GamePanel 第 80–93 行:
addMouseMotionListener(new MouseAdapter() {
    @Override
    public void mouseMoved(MouseEvent e) {        // L82
        mouseX = e.getX();  mouseY = e.getY();    // 记录鼠标位置(画准星用)
        controller.setAim(e.getX(), e.getY());     // 告诉控制器鼠标在哪 → 算瞄准角度
    }
    @Override
    public void mouseDragged(MouseEvent e) { /* 同上 */ }   // L89,按住拖动也算
});
```

- `MouseMotionListener` = 鼠标**移动**事件(`mouseMoved` = 没按键移动、`mouseDragged` = 按着键拖)。
- `controller.setAim(mx, my)` 里(在 [`GameController` 第 102 行](../src/main/java/com/game/game/GameController.java#L102)):`angle = atan2(my - player.y, mx - player.x)` —— 鼠标相对玩家的角度。
- **这是「事件驱动」**:鼠标一动就**立刻**更新角度(不等下一帧),瞄准是**即时**的。

### 3.2 鼠标点击 → 射击(MouseListener)

> 📍 [`GamePanel.java` · `mousePressed` 第 96–106 行](../src/main/java/com/game/game/GamePanel.java#L96) · [`isInExitButton` 第 371 行](../src/main/java/com/game/game/GamePanel.java#L371)

```java
// GamePanel 第 96–106 行:
addMouseListener(new MouseAdapter() {
    @Override
    public void mousePressed(MouseEvent e) {       // L98
        if (isInExitButton(e.getX(), e.getY())) {   // L99,先判:点的是右上角"设置"按钮?
            if (onSettings != null) onSettings.run();
            return;
        }
        controller.shoot();                          // L105,否则 = 射击
    }
});
```

- `MouseListener` = 鼠标**点击**事件(`mousePressed` = 按下瞬间)。
- 先 hit-test「设置」按钮([`isInExitButton` L371](../src/main/java/com/game/game/GamePanel.java#L371),手动画在画布右上角的一个矩形),点中就打开设置菜单;否则才是射击。
- **也是事件驱动**:点一次触发一次 shoot。

### 3.3 准星

> 📍 [`GamePanel.java` · `drawCrosshair` 第 303 行](../src/main/java/com/game/game/GamePanel.java#L303)

`mouseX / mouseY`(L53-54)记录最新鼠标位置,`paintComponent` 末尾 [`drawCrosshair(g)` L303](../src/main/java/com/game/game/GamePanel.java#L303) 在那个位置画准星圆环。

## 四、键盘(轮询)vs 鼠标(事件)—— 两种模式对照 ⭐

| | 键盘(WASD) | 鼠标(瞄准/射击) |
|---|---|---|
| 何时响应 | **每帧**(tick 轮询 Set) | **事件瞬间**(`mouseMoved`/`mousePressed`) |
| 监听器 | `KeyListener`(L114) | `MouseMotionListener`(L80)+ `MouseListener`(L96) |
| 数据结构 | `Set<Integer>`(L50) | 直接用事件坐标 |
| 为什么 | 需要**连续移动**(按住=持续动) | **即时反应**(动鼠标=立刻转枪口) |

> **关键洞察**:同一个项目里,键盘用**轮询(Set + tick)**、鼠标用**事件驱动**——因为两种输入需求不同:移动要持续(轮询),瞄准要即时(事件)。

---

## 五、输入 → 游戏的完整链路

```
键盘: WASD → pressedKeys(Set,L50) → moveDirX/Y(L183/195) → controller.setMoveDir(L139) → update() 第 0 步移动玩家
鼠标移动: → mouseMoved(L82) → controller.setAim(算 angle) → shoot()/draw 用 angle
鼠标点击: → mousePressed(L98) → controller.shoot()(创建子弹)
```

**输入不直接改对象位置 / 角度**,而是喂给 `GameController`(`setMoveDir` / `setAim` / `shoot`),由 controller 在 `update()` 里统一应用。**输入归面板、逻辑归控制器**——又是关注点分离。

## 六、别处的输入(按钮点击)

除了游戏里的键盘 / 鼠标,界面的**按钮**(登录 / 注册 / 菜单等)用 `ActionListener`:

> 📍 例如 [`LoginFrame.java` · 登录按钮 第 106 行](../src/main/java/com/game/ui/LoginFrame.java#L106)

```java
loginButton.addActionListener(e -> doLogin());   // 点按钮 → 调 doLogin
```

这也是事件驱动——点一下触发一次。和游戏的鼠标点击同理,只是 `ActionListener` 是按钮专用监听器。

## 七、自测

1. **为什么键盘用 Set 存按住的键,不用单个变量?**
2. **为什么键盘用「轮询(tick 读 Set)」而不在 keyPressed 里直接移动?**
3. **失焦时 `pressedKeys.clear()` 防什么问题?**
4. **鼠标移动(mouseMoved)是轮询还是事件驱动?为什么不用轮询?**
5. **JPanel 默认能收键盘事件吗?要做什么?**
6. **输入(键盘/鼠标)为什么不直接改 Player 位置,而是喂给 controller?**

<details><summary>答案</summary>

1. Set 能同时记多个键(W+D = 斜走);单个变量按 D 会覆盖 W,丢了斜走。
2. keyPressed 按住只触发一次(不是每帧),直接移动只动一下。轮询 Set = 每帧检查 → 按住持续动。
3. 防「卡键」:按住 W 后 Alt+Tab,`keyReleased` 可能不触发,W 永远在 Set 里 → 角色一直跑。clear 掉。
4. **事件驱动**。鼠标移动是连续高频事件,每动一次就更新角度,反应即时;轮询反而多余(没有"每帧检查鼠标在哪"的必要)。
5. 不能。JPanel 默认 `focusable = false`。要 `setFocusable(true)` + `requestFocusInWindow()`。
6. 关注点分离:GamePanel 管输入(收集事件),GameController 管逻辑(怎么移动/瞄准/射击)。输入不直接改对象,而是通过 `setMoveDir/setAim/shoot` 告诉 controller,由 update 统一应用——便于测试(GameController 脱离界面也能跑)。

</details>
