## 一、问题

鼠标瞄准类窗口游戏的经典痛点:打的时候鼠标一滑出窗口,就点到了桌面/别的程序,游戏还失焦。再叠加之前发现的「全屏后游戏世界不适配」(世界尺寸是 `GameController` 里写死的 `WIDTH=800/HEIGHT=600` 常量),体验不好。

## 二、三处改动

### ① 鼠标锁定:滑出画布即推回出界点内侧([`GamePanel`](../../src/main/java/com/game/game/GamePanel.java))

新增一个 `java.awt.Robot` 字段(懒加载,`AWTException` 拿不到就降级为 null)。在已有的 `MouseListener` 里加 `mouseExited`:

```java
@Override
public void mouseExited(MouseEvent e) {
    grabMouseBack(e);
}

private void grabMouseBack(MouseEvent e) {
    if (robot == null || !timer.isRunning()) return;        // 只在游戏跑着时锁
    Window win = SwingUtilities.getWindowAncestor(this);
    if (win == null || !win.isActive()) return;             // 切到别的窗口:别抢鼠标
    // 推回出界点内侧 margin 像素:把出界坐标钳进画布、离边 margin,保留瞄准方向
    int margin = 8;
    int cx = Math.max(margin, Math.min(getWidth() - margin, e.getX()));
    int cy = Math.max(margin, Math.min(getHeight() - margin, e.getY()));
    Point p = new Point(cx, cy);
    SwingUtilities.convertPointToScreen(p, this);           // 画布坐标 → 屏幕坐标
    robot.mouseMove(p.x, p.y);
}
```

> 鼠标物理上出不去窗口 → 点不到别的程序。比「瞬移到中心」更柔和——**保留玩家原本的瞄准方向**,只在边界处轻微反弹(把出界点钳回到离边 8px 内侧)。`Robot.mouseMove` 吃**屏幕坐标**,所以要先用 `convertPointToScreen` 把画布坐标换算过去。
>
> 最初版本是「拉回中心」,玩家反映手感太跳(瞄着边缘、鼠标一滑准星就跳到正中),遂改成这个「推回内侧」版本。

### ② 隐藏系统光标(只留自绘准星)

画布上设一个**透明光标**,这样玩家只看到 `drawCrosshair()` 画的那枚准星,不再「系统箭头 + 自绘准星」叠两个:

```java
setCursor(getToolkit().createCustomCursor(
        new BufferedImage(1, 1, BufferedImage.TYPE_INT_ARGB),   // 1×1 全透明图当光标
        new Point(0, 0), "blank"));
```

> 透明光标只作用于画布;鼠标出了画布范围会自动恢复系统箭头,**不影响点别的窗口**。「设置/暂停」模态弹窗是独立组件,有自己的默认光标,不受画布 blank 影响——照样能点按钮。

### ③ 窗口不可缩放([`GameWindow`](../../src/main/java/com/game/ui/GameWindow.java))

```java
setResizable(false);   // 锁窗口尺寸:画布固定 800×600,防拉大后世界尺寸不匹配
```

顺带修了「全屏不适配」那条:窗口拉不动了,就不存在「窗口变大、世界还是 800×600」的错位。

## 三、关键设计点(都是「不能一直锁」)

- **只在游戏运行时锁**(`timer.isRunning()`):打开「设置」暂停菜单、本局结束时 `timer.stop()` → 自动放开,不困住鼠标。
- **切到别的窗口不抢**(`win.isActive()`):用户 Alt+Tab 切走时窗口失活 → 不拉回,否则会和用户抢鼠标。
- **两处降级,绝不崩**:`Robot` 拿不到(个别环境)→ 不锁但游戏正常;自定义光标不支持 → 保留默认光标。

## 四、踩坑 / 收获

- **「锁鼠标」必须有开关**:不能一进来就死锁,要按「游戏状态 + 窗口是否活动」动态开关,否则暂停/切窗时用户被鼠标困住,很糟糕。这两道闸(`timer.isRunning()` + `win.isActive()`)是核心。
- **`Robot.mouseMove` 吃屏幕坐标**:画布中心是 `(getWidth/2, getHeight/2)`,但要 `SwingUtilities.convertPointToScreen` 转成屏幕坐标才能用,新手常踩。
- **隐藏光标 = 透明 1×1 BufferedImage 当 customCursor**:Swing 没有「隐藏光标」的直接 API,这是通用套路。
- **`setResizable(false)` 是最省事的「自适应」**:与其让逻辑层跟上窗口尺寸(改动大、还得重新平衡难度数值),不如直接锁死窗口,小游戏常用做法。
- **IDE 诊断有「中间态」**:这次一连串 Edit 过程中,IDE 报过 `grabMouseBack 未定义`(因为引用先于定义应用)、若干 `import 未使用`——都是**编辑过程中的快照**,以 `javac` 编译结果为准,别被中间告警带偏。
