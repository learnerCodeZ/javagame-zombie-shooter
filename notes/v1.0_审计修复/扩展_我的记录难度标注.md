# 扩展复盘:我的记录 = 全部战绩 + 难度列(按时间倒序)

| 项 | 内容 |
|---|---|
| 起因 | 用户反馈:"我的记录"里看不出每条是简单还是困难。原来的"我的记录"是**按当前难度过滤**的(只看到一种难度的战绩),且难度只标在顶栏小标签 |
| 改动 | 「我的记录」改为展示**当前用户全部战绩**(不分难度),表格加**「难度」列**逐行标注简单/困难,按**时间倒序**(最近发生的在最上) |
| 状态 | ✅ 完成、编译通过、TestDao 运行通过 |
| 日期 | 2026-07-12 |
| 文件 | [`dao/RecordDao.java`](../../src/main/java/com/game/dao/RecordDao.java)、[`ui/LeaderboardFrame.java`](../../src/main/java/com/game/ui/LeaderboardFrame.java)、[`TestDao.java`](../../src/main/java/com/game/TestDao.java) + 6 份正式文档 + 答辩 00 |

---

## 一、设计决策

| 决策点 | 选择 | 理由 |
|---|---|---|
| 我的记录范围 | **全部战绩**(不分难度) | "我的记录"语义=个人历史,应能看到自己所有难度的对局 |
| 排序 | **`record_time DESC`**(最近在上) | 个人历史按时间看最自然;和榜单(按分数)区分开 |
| 难度标注 | 表格加**「难度」列**(简单/困难) | 每条逐行标注,比顶栏标签直观 |
| 难度列显示范围 | **三档都显示**(简单榜/困难榜/我的记录) | 列结构统一;在简单榜里全是"简单"也无害,还能强化"你在看哪个榜" |

> 简单榜/困难榜仍是**全部玩家、按难度过滤、按分数倒序**(`topN`),没动。只改了「我的记录」。

## 二、代码改动(3 个文件)

### 1. [`RecordDao.mine`](../../src/main/java/com/game/dao/RecordDao.java) —— 签名 + SQL

```java
// 改前:按难度过滤、按分数倒序
public List<GameRecord> mine(int userId, String difficulty) {
    ... WHERE user_id = ? AND difficulty = ? ORDER BY score DESC ...

// 改后:不分难度、按时间倒序
public List<GameRecord> mine(int userId) {
    ... WHERE user_id = ? ORDER BY record_time DESC ...
```

`map()` 本就读 `difficulty` 列,**不用动**;`GameRecord` 也早有 `difficulty` 字段,**不用动**。

### 2. [`LeaderboardFrame`](../../src/main/java/com/game/ui/LeaderboardFrame.java) —— 4 处

- **表格列**:加「难度」列 → `{"排名","昵称","分数","难度","击杀","存活(秒)","时间"}`(难度放分数后)。
- **`reload()`**:mineMode 改调 `dao.mine(userId)`(不再传难度);每行加 `diffLabel(r.getDifficulty())`。
- **`updateViewLabel()`**:mineMode 标签改「当前:我的记录(全部 · 按时间倒序)」;非 mineMode 仍「当前:简单榜/困难榜」。
- **新增 `diffLabel()`**:把 DB 串 `EASY/HARD` 转 `简单/困难`(用 `Difficulty.valueOf(...).label`,空/未知兜底)。

### 3. [`TestDao`](../../src/main/java/com/game/TestDao.java)

`mine(userId, "EASY")` → `mine(userId)`(签名变了,1 处调用同步)。

- **javac 全量编译通过**。

## 五、踩坑 / 收获

- **改一个方法签名牵动一片**:`mine` 从 `(userId, difficulty)` → `(userId)`,牵动 DAO 自己 + UI 调用 + TestDao 调用 + UML + 典型 SQL + 流程图 + 多份功能描述。**先 grep 调用点再改**,避免漏。
- **语义变化要同步文档**:这次"我的记录"从"按当前难度过滤"变成"全部战绩",是**语义变了**(不是单纯加列),所以文档里所有"叠加只看自己/在当前难度上"的描述都得改——光改代码不改文档,文档就说谎了。
- **「全部+难度列」比「动态显隐列」省事**:难度列在简单榜里虽全是同一个值,但统一显示比"只在我的记录显示该列"实现简单得多(不用动态改表模型),权衡后选了统一。
