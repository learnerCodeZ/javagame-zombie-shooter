## 二、项目背景与意义

参考原型是一款适合少儿编程学习的 Scratch 射击小游戏，玩法是"鼠标瞄准 → 点击射击 → 消灭僵尸"。它简单有趣、视觉友好，非常适合作为 Java 课设的载体：玩法不复杂，能把面向对象（玩家、僵尸、子弹三类对象）、游戏主循环、碰撞检测、事件驱动等知识点完整地体现出来；再加上登录、注册、成绩存档、排行榜，又能覆盖数据库设计（建表、增删改查、表间关联）的考核点。

一个项目同时满足"Java 程序设计"与"数据库原理与应用"两门课，是课设的理想选题。

## 四、需求分析

### 4.1 功能需求

**F1 用户模块**
- F1.1 注册：手机号、密码、昵称，密码不少于 6 位
- F1.2 登录：校验手机号 + 密码，失败提示
- F1.3 退出登录

**F2 主菜单模块**
- 开始游戏、排行榜、游戏说明、设置（音效开关/难度）、退出

**F3 游戏模块（核心）**
- F3.1 玩家角色固定/可移动，鼠标瞄准，点击发射子弹
- F3.2 僵尸定时刷新，向玩家移动
- F3.3 子弹击中僵尸：僵尸消失、得分增加、击杀数 +1
- F3.4 僵尸碰到玩家：扣血，血量归零游戏结束
- F3.5 HUD：得分、击杀数、剩余血量、当前时间
- F3.6 游戏结束后把成绩写入数据库

**F4 排行榜模块**
- 按分数倒序展示前 N 名，显示昵称、分数、击杀数、用时、日期
- 按难度分「简单榜 / 困难榜」(按分数倒序)；「我的记录」看自己的全部战绩(按时间倒序、含难度标注)

**F5 数据管理**
- 注册写库、登录查库、成绩写库、排行榜查库

### 4.2 非功能需求

- 界面友好、卡通风格，操作简单（鼠标 + 少量键盘）
- 游戏帧率稳定（约 30~60 FPS，用 `javax.swing.Timer` 控制约 16~33ms 一帧）
- 数据库连接失败有友好提示，不崩溃
- 代码风格规范、有注释，便于答辩讲解

### 5.2 模块划分（共 7 个模块）

1. **启动模块** `GameApp`：程序入口，初始化外观主题，打开登录窗体。
2. **用户模块** `LoginFrame / RegisterFrame`：注册、登录（注册即成功，`role=user`，无需审核）。
3. **菜单模块** `MainFrame`：主菜单与功能跳转，含"修改密码"入口；`role=admin` 时显示"用户管理"入口。
4. **游戏模块** `GameWindow + GamePanel + GameController + Player/Zombie/Bullet`：核心玩法。
5. **排行榜模块** `LeaderboardFrame`：查询并展示成绩。
6. **用户管理与账号安全模块** `AdminFrame / ChangePasswordDialog`：基于 `admin` 角色的后台管理——查看所有用户、审核忘记密码重置申请（通过则重置为 `123456` / 拒绝）、删除用户（admin 账号不可删、不可被重置）；所有用户均可修改密码；登录窗体提供"忘记密码？"申请入口（同一用户已有 `pending` 申请时不可重复提交）。
7. **数据访问模块** `DBUtil / UserDao / RecordDao / ResetRequestDao`：封装所有 SQL 操作。

## 六、详细设计

### 6.1 游戏玩法设计

- **场景**：固定大小游戏区（如 800×600），浅色背景。
- **玩家**：橙色猫角色，位于画面中下，跟随鼠标方向旋转枪口；点击鼠标左键发射子弹。
- **敌人**：绿色僵尸，每隔一段时间从画面边缘随机位置生成，直线向玩家移动。
- **子弹**：从枪口沿瞄准方向匀速飞行，超出画面销毁。
- **碰撞**：子弹与僵尸圆形碰撞检测；僵尸与玩家圆形碰撞检测。
- **得分**：击杀一个僵尸 +10 分（可按难度调整）。
- **血量**：玩家初始 100，被僵尸碰到 -20，归零即 Game Over。
- **结束**：弹出结算面板，显示分数/击杀数/用时，并写入数据库。

```
        │ listAllUsers()   │
        │ deleteUser()     │
        └──────────────────┘
```

**关键类与类间关系说明**：
- `Player / Zombie / Bullet`：可抽象出接口 `GameObject`（含 `update()`、`draw(Graphics)`），体现**多态**。
- `User 1 ──< * GameRecord`：一个用户有多条游戏记录，**一对多**，外键 `user_id`。
- `User 1 ──< * PasswordResetRequest`：一个用户可有多条密码重置申请，**一对多**，外键 `user_id`。
- `GameController` 依赖 `RecordDao`：游戏结束时把成绩落库。
- `DBUtil` 被 `UserDao / RecordDao / ResetRequestDao` 共用：封装 JDBC 连接获取/释放。
- **扩展模块类**：`User` 含 `role` 字段（`admin` / `user`）及 `getRole()/setRole()`；`UserDao` 扩展 `changePassword / listAllUsers / deleteUser`；新增 `PasswordResetRequest`、`ResetRequestDao`、`AdminFrame`、`ChangePasswordDialog`。完整类图见《详细设计说明书》E.3。

#### 6.3.2 表结构

**user（用户表）**

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 主键 |
| phone | VARCHAR(11) | NOT NULL, UNIQUE | 登录手机号 |
| password | VARCHAR(64) | NOT NULL | 密码（建议 MD5 存储） |
| nickname | VARCHAR(50) | | 昵称 |
| role | VARCHAR(10) | NOT NULL DEFAULT 'user' | 角色（admin/user） |
| create_time | DATETIME | DEFAULT NOW() | 注册时间 |

**game_record（游戏记录表）**

| 字段 | 类型 | 约束 | 说明 |

| id | INT | PK, AUTO_INCREMENT | 主键 |
| user_id | INT | NOT NULL, FK→user.id) | 外键，所属用户 |

| kill_count | INT | DEFAULT 0 | 击杀数 |
| survive_sec | INT | DEFAULT 0 | 存活秒数 |
| record_time | DATETIME | DEFAULT NOW() | 记录时间 |

**password_reset_request（密码重置申请表）**

| 字段 | 类型 | 约束 | 说明 |
|---|---|---|---|
| id | INT | PK, AUTO_INCREMENT | 主键 |

## 九、代码规范（锯齿型书写格式）

为保证"锯齿型书写格式"与可读性，统一约定：

- **缩进**：统一 4 个空格，禁止 Tab 与空格混用。
- **花括号**：采用 K&R 风格（`{` 不换行，`}` 单独成行）。
- **命名**：类名 PascalCase；方法/变量 camelCase；常量全大写 + 下划线。
- **包名**：全小写，`com.game.xxx`。
- **注释**：类、public 方法用 Javadoc；复杂逻辑加行内注释。
- **文件编码**：UTF-8（含中文字符串资源）。

示例（注意锯齿状缩进层次）：

```java
public class Player implements GameObject {
    private int x;
    private int y;
    private int hp = 100;

    public void hit(int dmg) {
        if (dmg <= 0) {
            return;
        }
        this.hp -= dmg;
        if (this.hp < 0) {
            this.hp = 0;
        }
    }
}
```
